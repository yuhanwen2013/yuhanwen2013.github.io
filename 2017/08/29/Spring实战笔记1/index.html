这是Spring实战读书笔记 第一部分
Spring 有连个关键特征 控制反转（IOC）和 面向切面编程（AOP）
当然IOC还有另外一种说法叫 依赖注入（DI）

简化Java开发

为了降低Java开发的复杂性，Spring采取了以下四种关键策略

  • 基于POJO的轻量级和最小侵入性编程。
  • 通过依赖注入和面向接口实现松耦合。
  • 基于切面和惯例进行声明式编程。
  • 通过切面和模板减少样板代码。

例如 A是个构造函数
public A(){
this.c = new B();
}

可以看出构造函数A中自行创建了B，这使得A和B紧密的耦合到了一起。这会限制函数的使用场景，并会增大单元测试的困难。

当然耦合具有两面性，一方面紧密耦合的代码难以测试，难以复用。但是一定程度的耦合是必要的，完全没有耦合的代码什么都做不了。
